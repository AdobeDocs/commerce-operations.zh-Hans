# 需求核对清单

以下问题可作为起点，以查看应记录哪些信息以确认组织是否已做好准备。 在制定RFP时，这些功能也会有所帮助。

## 商业

- 新电子商务平台的业务目标是什么？

- 更改您当前的电子商务平台的原因是什么？

- 网站基础架构的目标是什么？

- 交付新电子商务平台的时间范围是多久？

- 将为此项目分配多少IT项目经理？

- 您的多少业务分析师将分配到此项目？

- 您的多少技术分析师将分配到此项目？

- 将为此项目分配多少HTML开发人员？

- 当前业务流程有哪些文档？

- 当前系统集成有哪些文档？

- 当前流程存在什么文档？

- 当前数据流有哪些文档？

- 由谁提供培训？

- 在上线之前将完成哪些培训？

- 哪些培训将在上线后完成？

- 上线后需要哪些Adobe Commerce支持？

- 该项目是否依赖于其他系统开发项目？

- 您如何看到您的销售在接下来的12-24个月中增长？

- 您的主要竞争对手是谁？ 请提供其网站的链接。 您希望如何使您的在线体验与竞争对手脱颖而出？

- 您认为未来业务增长将来自何处？

- 数字商务在您的业务策略中发挥什么作用？ 您设置此电子商务平台的主要目标是什么？

- 您是否有任何品牌/公司可作为您如何发展全渠道业务的参考？

- 哪些团队或个人在推动电子商务战略？ 描述相关职位。

## 当前平台

- 如何托管当前平台：内部、托管提供商、专用云服务器或托管云服务器？

- 当前平台具有哪些环境：开发、QA、预生产、生产？

- 开发环境中有多少个Web服务器和数据库服务器？

- QA环境中有多少个Web服务器和数据库服务器？

- 暂存（预生产）环境中有多少Web服务器和数据库服务器？

- 生产环境中有多少Web服务器和数据库服务器？

- 它是否为具有负载平衡的多服务器体系结构？

- 它是高可用性系统体系结构吗？

- 您当前的网站遇到了什么问题？

- 当前目录大小（SKU数量）？

- 每天平均访客数？

- 每小时平均并发会话数？

- 每天的平均页面查看次数？

- 每小时和每天的平均订单数？

## 预期的平台要求

- 您将使用哪个Adobe Commerce版本？

- 未来平台将如何托管：内部、托管提供商、专用云服务器或托管云服务器？

- 未来平台将具有哪些环境：开发、QA、预生产、生产？

- 开发环境中将有多少Web和数据库服务器？

- 它是否为高可用性系统体系结构？

- 您将有多少位Adobe Commerce管理员？

- 预期的目录大小（SKU数量）？

- 预计每日平均访客数？

- 每小时的平均并发会话数？

- 预计每日平均页面查看次数？

- 应将哪些数据从旧站点导入新站点？ （例如：产品、客户、订单历史记录）

## 网站

- 将实施多少国内网站？

- 将实施哪些语言？

- 将实施多少个国际网站？

- 网站支持哪些地区？

- 将实施哪些语言？

- 将采用哪些货币？

- 您是否拥有适用于每个国家/地区的交付服务级别协议？

- 贵机构的每个国家/地区哪些服务提供商？

- 每个国家/地区的会计提供商是谁？

- 是否需要自定义国际网站主题？

- 这主要是B2C站点还是B2B站点？ 是否有任何B2B2B或B2B2C元素？

- 是否已有设计可供改装，或者平台是否将从头开始设计？

- 是否需要使用Headless商务（分隔的前端和后端层）？

- 响应式设计的断点（平板电脑/移动设备）是什么？

- 是否需要移动设备应用程序？ 是否应将PWA用于移动前端？

- 应该测试的任何特定浏览器（标准浏览器IE9+、Firefox、Chrome、Safari除外）？

- 每个前端使用什么语言？ 翻译的内容是否可用或是否需要支持？

- 是否有多个网站？ 如果是这样的话，客户能否在所有网站上使用其凭据？

- 产品数据是否在所有站点之间共享？

- 网站之间的设计是否发生了更改？

- 是否有订阅选项以及如何管理订阅者？

- 是否有特定的SEO要求？ 如何管理搜索引擎营销？

## 集成

- 哪种CMS系统将与Adobe Commerce集成？ （示例： WordPress、Drupal、Concrete5）

是否存在可以使用的API？

- 是否为此第三方系统集成设计和开发了系统错误处理？

- 哪些ERP系统将与Adobe Commerce集成？ （示例： SAP、MS Dynamics NAV）

- 哪一种承运人系统将与Adobe Commerce集成？

- 哪种税务软件系统将与Adobe Commerce集成？ （例如：Taxware）

- 产品数据将从哪个系统导入Adobe Commerce？

- 导入的产品数据加载频率？

- Adobe Commerce会将产品数据导出到哪个系统中？

- 导出的产品数据加载的频率？

- 订单数据将从哪个系统导入Adobe Commerce？

- 导入订单数据加载的频率？

- Adobe Commerce会将订单数据导出到哪个系统中？

- 导出的订单数据加载的频率？

- 使用哪个分析平台？ (示例： Adobe Analytics、Google Analytics)

- 是否会进行单点登录和社交网络共享？ 如果是，那么哪些社交网络？

- 是否有任何奖励或忠诚计划？ 是否与POS系统集成以提供奖励？

- 如何处理产品退货？ 客户是否应该在线记录退货请求？

- 是否需要在线聊天工具？ 是否需要聊天机器人？

- 您希望您的网站具有哪个支付网关？

- 哪种订单管理系统将与Adobe Commerce集成？ (示例： Microsoft Dynamics、SAP、Retail Pro)

- 哪种产品信息管理系统将与Adobe Commerce集成？ （示例： Akeneo、InRiver、Bluestone）

- 哪种客户关系管理系统将与Adobe Commerce集成？ （示例： Hubspot、Salesforce、Klaviyo）
