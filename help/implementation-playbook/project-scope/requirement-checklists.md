# 要求核对表

以下问题可作为一个起点，用于查看应记录哪些信息以确认组织准备。 在开发RFP时，它们也会有所帮助。

## 商业

- 新电子商务平台的业务目标是什么？

- 更改当前电子商务平台的原因是什么？

- 网站基础结构的目标是什么？

- 您需要多长时间来交付新的电子商务平台？

- 您有多少IT项目经理将被分配到此项目？

- 将为此项目分配多少业务分析员？

- 将为此项目分配多少个技术分析员？

- 您有多少HTML开发人员将分配到此项目？

- 当前业务流程有哪些文档？

- 当前系统集成有哪些文档？

- 当前流程有哪些文档？

- 当前数据流有哪些文档？

- 谁将提供培训？

- 在上线之前将完成哪些培训？

- 上线后将完成哪些培训？

- 上线后需要哪些Adobe Commerce支持？

- 此项目是否依赖于其他系统开发项目？

- 您如何看到未来12-24个月的销售额增长？

- 谁是您的主要竞争对手？ 请提供其网站的链接。 您希望如何将在线体验与竞争对手区分开来？

- 您认为未来业务增长来自何处？

- 数字商务在您的业务策略中起着什么作用？ 设置此电子商务平台的主要目标是什么？

- 您有哪些品牌/公司可作为您如何发展全渠道业务的参考？

- 哪些团队或个人在推动电子商务策略？ 描述相关立场。

- 生产环境中有多少台Web服务器和数据库服务器？

- 它是否是具有负载平衡的多服务器架构？

- 它是高可用性系统架构吗？

- 您当前的网站遇到哪些问题？

- 当前目录大小（SKU数量）？

- 平均每日访客数？

- 每小时平均并发会话数？

- 平均每日页面查看次数？

- 每小时和每天的平均订单数？

## 网站

- 将实施多少个国内网站？

- 每个前端的语言是什么？ 翻译的内容是否可用或是否需要支持？

- 是否在所有网站之间共享产品数据？

- 设计在网站之间是否发生了更改？

- 是否有订阅选项以及如何管理订阅者？

- 是否有特定的SEO要求？ 如何管理搜索引擎营销？
