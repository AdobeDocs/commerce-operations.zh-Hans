# ACSD-69331：对于只有&#x200B;**[!UICONTROL Create Folder]**&#x200B;权限的管理员用户，创建媒体集文件夹失败

ACSD-69331修补程序修复了仅授予&#x200B;**[!UICONTROL Create Folder]**&#x200B;权限时，管理员用户无法在CMS媒体集中创建文件夹的问题。 修复后，他们可以按预期创建文件夹。 安装[[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.76时，此修补程序可用。 修补程序ID为ACSD-69331。 请注意，此问题计划在Adobe Commerce 2.4.9中修复。

## 受影响的产品和版本

**为Adobe Commerce版本创建了修补程序：**

* Adobe Commerce（所有部署方法） 2.4.7-p8

**与Adobe Commerce版本兼容：**

* Adobe Commerce（所有部署方法） 2.4.5 - 2.4.8-p3

>[!NOTE]
>
>该修补程序可能适用于具有新[!DNL Quality Patches Tool]发行版本的其他版本。 要检查修补程序是否与您的Adobe Commerce版本兼容，请将`magento/quality-patches`包更新到最新版本，并在[[!DNL Quality Patches Tool]：搜索修补程序页面](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html)上检查兼容性。 使用修补程序ID作为搜索关键字来查找修补程序。

## 问题

在媒体集中仅具有&#x200B;**[!UICONTROL Create Folder]**&#x200B;权限的管理用户无法创建文件夹，除非同时授予了&#x200B;**[!UICONTROL Delete Folder]**&#x200B;权限。

<u>重现步骤</u>：

1. 登录到“管理”面板。
1. 创建有权访问&#x200B;**[!UICONTROL Dashboard]**&#x200B;和&#x200B;**[!UICONTROL Content]** > **[!UICONTROL Elements]** > **[!UICONTROL Media Gallery]**&#x200B;的新角色，仅授予&#x200B;**[!UICONTROL Create Folder]**&#x200B;权限（不授予&#x200B;**[!UICONTROL Delete Folder]**）。
1. 创建新的管理员用户并分配此角色。
1. 以新的管理员用户身份登录。
1. 转到&#x200B;**[!UICONTROL Content]** > *[!UICONTROL Media]* > **[!UICONTROL Media Gallery]**。
1. 选择目录（例如，`catalog`）。
1. 单击&#x200B;**[!UICONTROL Create Folder]**。
1. 输入并确认文件夹名称。

<u>预期的结果</u>：

* 文件夹创建仅对具有&#x200B;**[!UICONTROL Create Folder]**&#x200B;权限的管理员用户有效。
* 创建文件夹不需要&#x200B;**[!UICONTROL Delete Folder]**&#x200B;权限。

<u>实际结果</u>：

* 用户无法创建仅具有&#x200B;**[!UICONTROL Create Folder]**&#x200B;权限的文件夹。
* 创建文件夹需要授予&#x200B;**[!UICONTROL Delete Folder]**&#x200B;权限。
