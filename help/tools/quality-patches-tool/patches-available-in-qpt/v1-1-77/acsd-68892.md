# ACSD-68892：可缓存页面的[!DNL Fastly]缓存行为不一致

ACSD-68892修补程序修复了无法从[!DNL Fastly]缓存正确存储或提供可缓存页面，从而导致缓存行为不一致和性能下降的问题。 安装[[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.77时，此修补程序可用。 修补程序ID为ACSD-68892。 请注意，此问题计划在Adobe Commerce 2.4.9中修复。

## 受影响的产品和版本

**为Adobe Commerce版本创建了修补程序：**

* Adobe Commerce（所有部署方法） 2.4.8-p2

**与Adobe Commerce版本兼容：**

* Adobe Commerce（所有部署方法） 2.4.8 - 2.4.8-p3

>[!NOTE]
>
>该修补程序可能适用于具有新[!DNL Quality Patches Tool]发行版本的其他版本。 要检查修补程序是否与您的Adobe Commerce版本兼容，请将`magento/quality-patches`包更新到最新版本，并在[[!DNL Quality Patches Tool]：搜索修补程序页面](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html?lang=zh-Hans)上检查兼容性。 使用修补程序ID作为搜索关键字来查找修补程序。

## 问题

未从[!DNL Fastly]缓存中正确存储或提供可缓存的页面，从而导致缓存行为不一致并降低了性能。

<u>重现步骤</u>：

1. 启用[!DNL Fastly]缓存。
1. 创建新的网站/商店/商店视图。
1. 将辅助网站代码添加到`index.php`：

```
$_SERVER['MAGE_RUN_TYPE'] = 'website';
$_SERVER['MAGE_RUN_CODE'] = 'web2';
```

1. 使用curl检查缓存控制标头：

```
curl -Is https://localhost/m247/pub/ | grep cache-control
```

<u>预期的结果</u>：

可缓存的页面缓存在[!DNL Fastly]中。

<u>实际结果</u>：

未在[!DNL Fastly]中缓存可缓存的页面，标头中返回了`max-age=0`。

```
curl -Is https://localhost/m24develop/pub/ |grep cache-control
cache-control: max-age=0, must-revalidate, no-cache, no-store
```
