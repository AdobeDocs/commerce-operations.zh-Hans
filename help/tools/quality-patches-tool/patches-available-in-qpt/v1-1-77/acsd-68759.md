# ACSD-68759：当[!UICONTROL Date of Birth]在店面可见时，在阿拉伯语（沙特阿拉伯）区域设置下创建客户帐户失败

The ACSD-68759 patch fixes the issue where the Customer **[!UICONTROL Create an Account]** page fails to load when the **[!UICONTROL Locale]** is set to *Arabic (Saudi Arabia)* and the **[!UICONTROL Date of Birth]** attribute is displayed on the storefront. 安装[[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.77时，此修补程序可用。 The patch ID is ACSD-68759. 请注意，此问题计划在Adobe Commerce 2.4.9中修复。

## 受影响的产品和版本

**为Adobe Commerce版本创建了修补程序：**

* Adobe Commerce (all deployment methods) 2.4.7-p5

**与Adobe Commerce版本兼容：**

* Adobe Commerce (all deployment methods) 2.4.4-p2 - 2.4.4-p16, 2.4.5-p1 - 2.4.8-p3

>[!NOTE]
>
>该修补程序可能适用于具有新[!DNL Quality Patches Tool]发行版本的其他版本。 要检查修补程序是否与您的Adobe Commerce版本兼容，请将`magento/quality-patches`包更新到最新版本，并在[[!DNL Quality Patches Tool]：搜索修补程序页面](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html?lang=zh-Hans)上检查兼容性。 使用修补程序ID作为搜索关键字来查找修补程序。

## 问题

The **[!UICONTROL Create an Account]** page fails to load when the **[!UICONTROL Locale]** is set to Arabic (Saudi Arabia) and the **[!UICONTROL Date of Birth]** attribute is visible on the storefront, resulting in a `TypeError`.

<u>重现步骤</u>：

1. Log in to the Admin.
2. Go to **[!UICONTROL Stores]** > *[!UICONTROL Attributes]* > **[!UICONTROL Customer]**.
3. Edit the **[!UICONTROL dob]** attribute code.
   * Set **[!UICONTROL Show on Storefront]** to *Yes*.
4. 单击&#x200B;**[!UICONTROL Save Attribute]**。
5. Go to **[!UICONTROL Stores]** > *[!UICONTROL Settings]* > **[!UICONTROL Configuration]** > **[!UICONTROL General]** > **[!UICONTROL Locale Options]**.
   * 将&#x200B;**[!UICONTROL Locale]**&#x200B;设置为&#x200B;*阿拉伯语（沙特阿拉伯）*。
6. 去店面。
7. 打开&#x200B;**[!UICONTROL Create an Account]**&#x200B;页面。

<u>预期的结果</u>：

The **[!UICONTROL Create an Account]** page loads without errors when the **[!UICONTROL Locale]** is set to *Arabic (Saudi Arabia)* and the **[!UICONTROL Date of Birth]** attribute is displayed on the storefront.

<u>实际结果</u>：

* The **[!UICONTROL Create an Account]** page does not load.
* The following error is logged:

  ```
  [2025-10-31T02:14:35.029229+00:00] report.CRITICAL: TypeError: iterator_to_array(): Argument #1 ($iterator) must be of type Traversable|array, null given in app/code/Magento/Customer/Block/Widget/Dob.php:421
  ```
