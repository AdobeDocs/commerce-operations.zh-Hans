# ACSD-68490：在可配置产品创建期间对受限管理员可见的&#x200B;**[!UICONTROL Add New Attribute]**&#x200B;按钮

ACSD-68490修补程序修复了在可配置产品创建期间受限管理员看到&#x200B;**[!UICONTROL Add New Attribute]**&#x200B;按钮的问题。 安装[[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.75时，此修补程序可用。 修补程序ID为ACSD-68490。 请注意，此问题计划在Adobe Commerce 2.4.9中修复。

## 受影响的产品和版本

**为Adobe Commerce版本创建了修补程序：**

* Adobe Commerce（所有部署方法） 2.4.6

**与Adobe Commerce版本兼容：**

* Adobe Commerce（所有部署方法） 2.4.6 - 2.4.6-p13

>[!NOTE]
>
>该修补程序可能适用于具有新[!DNL Quality Patches Tool]发行版本的其他版本。 要检查修补程序是否与您的Adobe Commerce版本兼容，请将`magento/quality-patches`包更新到最新版本，并在[[!DNL Quality Patches Tool]：搜索修补程序页面](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html?lang=zh-Hans)上检查兼容性。 使用修补程序ID作为搜索关键字来查找修补程序。

## 问题

在创建可配置产品的过程中，**[!UICONTROL Add New Attribute]**&#x200B;按钮对受限管理员可见。

<u>重现步骤</u>：

1. 创建其他网站、商店和商店视图。
1. 创建仅对新网站具有访问权限的管理员角色并分配所有资源。
1. 创建受限管理员用户并将新角色分配给此用户。
1. 以受限用户身份登录到管理员。
1. 创建新的可配置产品。


<u>预期的结果</u>：

**[!UICONTROL Create New Attribute]**&#x200B;按钮对受限管理员用户不可用。

<u>实际结果</u>：

此时将显示404页面。
