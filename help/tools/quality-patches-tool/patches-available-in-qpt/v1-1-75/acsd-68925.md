# ACSD-68925：GraphQL响应现在遵循HTTP规范，对于无法解析、未授权或无效的请求返回4XX

此补丁可确保GraphQL在请求无法解析、未授权或以其他方式无效时返回相应的HTTP状态代码。 安装[[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.75时，此修补程序可用。 修补程序ID为ACSD-68925。 请注意，此问题计划在Adobe Commerce 2.4.9中修复。

## 受影响的产品和版本

**为Adobe Commerce版本创建了修补程序：**

* Adobe Commerce（所有部署方法） 2.4.8-p3

**与Adobe Commerce版本兼容：**

* Adobe Commerce（所有部署方法） 2.4.8 - 2.4.8-p3

>[!NOTE]
>
>该修补程序可能适用于具有新[!DNL Quality Patches Tool]发行版本的其他版本。 要检查修补程序是否与您的Adobe Commerce版本兼容，请将`magento/quality-patches`包更新到最新版本，并在[[!DNL Quality Patches Tool]：搜索修补程序页面](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html)上检查兼容性。 使用修补程序ID作为搜索关键字来查找修补程序。

## 问题

以前，无效的GraphQL请求会生成内部服务器错误，而不是GraphQL通过HTTP规范定义的正确&#x200B;*4XX*&#x200B;响应。

<u>重现步骤</u>：

使用未知/不正确的筛选器名称发送GraphQL请求。

```graphql
  {
    cmsBlocks(identifiers: ["home"]) {
    items {
      title
      }
    }
  }
```

<u>预期的结果</u>：

当无法分析请求、请求未获得授权或请求存在其他一般问题时返回&#x200B;*4XX*&#x200B;响应代码。 如果请求已解析并可处理，则应返回&#x200B;*200*&#x200B;响应代码。

<u>实际结果</u>：

在`exception.log`中返回了带有以下错误的&#x200B;*内部服务器错误*：

```text
report.CRITICAL: LogicException: Config element "FindFilterType" is not declared in GraphQL schema in lib/internal/Magento/Framework/GraphQl/Config.php:57
```
