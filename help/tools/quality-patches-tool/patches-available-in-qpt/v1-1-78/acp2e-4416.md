# ACP2E-4416：在Admin中创建客户奖励点时不会初始化

ACP2E-4416修补程序修复了在管理员中创建客户奖励点时未初始化的问题。 安装[[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.78时，此修补程序可用。 修补程序ID为ACP2E-4416。 请注意，此问题计划在Adobe Commerce 2.4.9中修复。

## 受影响的产品和版本

**为Adobe Commerce版本创建了修补程序：**

* Adobe Commerce（所有部署方法） 2.4.8-p3

**与Adobe Commerce版本兼容：**

* Adobe Commerce（所有部署方法） 2.4.4 - 2.4.8-p4

>[!NOTE]
>
>该修补程序可能适用于具有新[!DNL Quality Patches Tool]发行版本的其他版本。 要检查修补程序是否与您的Adobe Commerce版本兼容，请将`magento/quality-patches`包更新到最新版本，并在[[!DNL Quality Patches Tool]：搜索修补程序页面](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html?lang=zh-Hans)上检查兼容性。 使用修补程序ID作为搜索关键字来查找修补程序。

## 问题

在Admin中创建客户奖励点时未初始化。

<u>重现步骤</u>：

1. 通过管理员或REST API创建客户。
1. 发送GraphQL请求以检索客户的奖励积分余额。
1. 查看响应。

<u>预期的结果</u>：

系统返回客户的奖励积分余额。

<u>实际结果</u>：

系统返回错误：

```
{"errors":[{"message":"Internal server error","locations":[{"line":5,"column":9}],"path":["customer","reward_points","balance","points"]}],"data":{"customer":{"reward_points":{"balance":null}}}}
```
