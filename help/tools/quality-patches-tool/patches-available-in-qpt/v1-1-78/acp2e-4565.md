# ACP2E-4565：使用X-Adobe-Company标头时，GraphQL公司查询返回&#x200B;*当前客户未获得授权*

ACP2E-4565修补程序修复了以下问题：公司GraphQL查询返回错误&#x200B;*当请求中同时包含X-Adobe-Company和Store标头时，当前客户未获得授权*，即使已将客户正确分配到指定的公司和商店也是如此。 安装[[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.78时，此修补程序可用。 修补程序ID为ACP2E-4565。 请注意，此问题计划在Adobe Commerce 2.4.9中修复。

## 受影响的产品和版本

**为Adobe Commerce版本创建了修补程序：**

* Adobe Commerce（所有部署方法） 2.4.8-p3

**与Adobe Commerce版本兼容：**

* Adobe Commerce（所有部署方法） 2.4.7-p3

>[!NOTE]
>
>该修补程序可能适用于具有新[!DNL Quality Patches Tool]发行版本的其他版本。 要检查修补程序是否与您的Adobe Commerce版本兼容，请将`magento/quality-patches`包更新到最新版本，并在[[!DNL Quality Patches Tool]：搜索修补程序页面](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html)上检查兼容性。 使用修补程序ID作为搜索关键字来查找修补程序。

## 问题

公司GraphQL查询返回&#x200B;*当请求中同时包含X-Adobe-Company和Store标头时，当前客户未获得授权*，即使已将客户正确分配到指定的公司和商店。

<u>重现步骤</u>：

1. 启用B2B公司。
1. 创建其他网站、商店和商店视图。
1. 创建两个公司（第一公司和第二公司）。
1. 创建分配给第二个网站的用户。
1. 将两个公司分配给用户。
1. 使用以下标头执行以下GraphQL查询：
   * `X-Adobe-Company: MQ==`
   * `Store: storeview2`

   ```graphql
   query Company {
     company {
       id
       legal_name
       name
     }
   }
   ```

<u>预期的结果</u>：

公司数据返回成功。

<u>实际结果</u>：

GraphQL响应返回授权错误：

```
{
    "errors": [
        {
            "message": "The current customer isn't authorized.",
            "extensions": {
                "category": "graphql-authorization"
            }
        }
    ]
}
```
