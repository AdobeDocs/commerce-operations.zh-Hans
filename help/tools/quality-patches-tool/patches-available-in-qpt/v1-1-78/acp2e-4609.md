# ACP2E-4609：当某些引号包含已删除的产品时，**[!UICONTROL My Quotes]**&#x200B;页不显示引号

ACP2E-4609修补程序修复了以下问题：当某些引号包含已删除的产品时，**[!UICONTROL My Quotes]**&#x200B;页面不显示引号。 安装[[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.78时，此修补程序可用。 修补程序ID为ACP2E-4609。 请注意，此问题计划在Adobe Commerce 2.4.9中修复。

## 受影响的产品和版本

**为Adobe Commerce版本创建了修补程序：**

* Adobe Commerce（所有部署方法） 2.4.7-p8

**与Adobe Commerce版本兼容：**

* Adobe Commerce（所有部署方法） 2.4.6 - 2.4.8-p4

>[!NOTE]
>
>该修补程序可能适用于具有新[!DNL Quality Patches Tool]发行版本的其他版本。 要检查修补程序是否与您的Adobe Commerce版本兼容，请将`magento/quality-patches`包更新到最新版本，并在[[!DNL Quality Patches Tool]：搜索修补程序页面](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html)上检查兼容性。 使用修补程序ID作为搜索关键字来查找修补程序。

## 问题

当某些引号包含已删除的产品时，**[!UICONTROL My Quotes]**&#x200B;页面不显示引号。

<u>先决条件</u>：

公司和B2B报价功能已启用。

<u>重现步骤</u>：

1. 创建公司。
1. 创建多个简单的产品。
1. 创建至少有两个选项的捆绑产品，然后选择&#x200B;**[!UICONTROL User Defined]**。 将简单产品分配给不同的捆绑包选项。
1. 以公司管理员身份登录到店面。
1. 使用相同选项但数量不同的捆绑产品两次添加到购物车。
1. 导航到购物车并创建报价。
1. 删除关联的简单产品。
1. 要重现问题，报价应标记为`trigger_recollect: 1`。

   对Adobe Commerce数据库运行以下SQL命令：

   ```
   UPDATE quote SET trigger_recollect = 1 WHERE entity_id = 1;
   ```

1. 转到&#x200B;**[!UICONTROL My Account]** > **[!UICONTROL My Quotes]**。

<u>预期的结果</u>：

**[!UICONTROL My Quotes]**&#x200B;网格应已成功呈现。

<u>实际结果</u>：

1. 未填充&#x200B;**[!UICONTROL My Quotes]**&#x200B;网格。
1. http响应(`negotiable_quote/ui/render/`)：

   ```
   {"error":"Requested quote is not found. Row ID: quoteId = 1","errorcode":"0"}
   ```

1. `exception.log`中的错误：

   ```
   report.CRITICAL: Magento\Framework\Exception\NoSuchEntityException: Requested quote is not found. Row ID: quoteId = 1 in app/code/Magento/NegotiableQuote/Model/NegotiableQuoteManagement.php:673
   ```
