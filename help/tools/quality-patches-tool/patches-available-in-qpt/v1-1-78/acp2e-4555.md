# ACP2E-4555: Telephone numbers that contain periods or forward slashes don&#39;t validate properly

The ACP2E-4555 patch fixes the issue where modern telephone numbers containing `.` or `/` don&#39;t validate. 安装[[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.78时，此修补程序可用。 The patch ID is ACP2E-4555. 请注意，此问题计划在Adobe Commerce 2.4.9中修复。

## 受影响的产品和版本

**为Adobe Commerce版本创建了修补程序：**

* Adobe Commerce（所有部署方法） 2.4.8-p3

**与Adobe Commerce版本兼容：**

* Adobe Commerce (all deployment methods) 2.4.8 - 2.4.8-p4

>[!NOTE]
>
>该修补程序可能适用于具有新[!DNL Quality Patches Tool]发行版本的其他版本。 要检查修补程序是否与您的Adobe Commerce版本兼容，请将`magento/quality-patches`包更新到最新版本，并在[[!DNL Quality Patches Tool]：搜索修补程序页面](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html?lang=zh-Hans)上检查兼容性。 使用修补程序ID作为搜索关键字来查找修补程序。

## 问题

Modern telephone numbers containing `.` or `/` don&#39;t validate.

<u>重现步骤</u>：

1. Log in as a customer on the storefront.
1. Add a new address to the address book with a telephone number, for example: `06.76.40.32.22` or `+43680/2149568`.
1. Save the address details.

<u>预期的结果</u>：

可以使用带句点和正斜线的现代电话格式。

<u>实际结果</u>：

Validation fails for the telephone field due to the `.` or `/` characters.
