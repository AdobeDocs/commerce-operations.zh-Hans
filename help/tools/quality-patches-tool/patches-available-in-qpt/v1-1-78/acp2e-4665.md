# ACP2E-4665：通过REST API请求时，产品库中带有视频的可配置产品的子产品不会显示

ACP2E-4665修补程序修复了当视频包含在产品库中时，REST API不会返回可配置产品的子产品的问题。 安装[[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.78时，此修补程序可用。 修补程序ID为ACP2E-4665。 请注意，此问题计划在Adobe Commerce 2.4.9中修复。

## 受影响的产品和版本

**为Adobe Commerce版本创建了修补程序：**

* Adobe Commerce（所有部署方法） 2.4.8-p3

**与Adobe Commerce版本兼容：**

* Adobe Commerce（所有部署方法） 2.4.8 - 2.4.8-p4

>[!NOTE]
>
>该修补程序可能适用于具有新[!DNL Quality Patches Tool]发行版本的其他版本。 要检查修补程序是否与您的Adobe Commerce版本兼容，请将`magento/quality-patches`包更新到最新版本，并在[[!DNL Quality Patches Tool]：搜索修补程序页面](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html?lang=zh-Hans)上检查兼容性。 使用修补程序ID作为搜索关键字来查找修补程序。

## 问题

当视频出现在产品库中时，REST API不会返回可配置产品的子产品。

<u>重现步骤</u>：

1. 将视频添加到可配置产品的子产品。
1. 向以下REST API端点发送请求：
   `rest/v1/configurable-products/<PARENT_CONFIGURABLE_PRODUCT_SKU>/children`.

<u>预期的结果</u>：

REST API返回有关产品的信息。

<u>实际结果</u>：

REST API返回以下错误：

```
Error: Call to a member function getVideoProvider() on null in vendor/magento/module-product-video/Model/Product/Attribute/Media/ExternalVideoEntryConverter.php:87" exception.
```
