# ACP2E-4540： Fotorama库无法正确加载，因此只有第一个附加的图像可见

ACP2E-4540修补程序修复了Fotorama库未正确加载，导致仅显示所连接的第一张图像的问题。 安装[[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.78时，此修补程序可用。 修补程序ID为ACP2E-4540。 请注意，此问题计划在Adobe Commerce 2.4.9中修复。

## 受影响的产品和版本

**为Adobe Commerce版本创建了修补程序：**

* Adobe Commerce（所有部署方法） 2.4.7-p7

**与Adobe Commerce版本兼容：**

* Adobe Commerce（所有部署方法） 2.4.7 - 2.4.8-p4

>[!NOTE]
>
>该修补程序可能适用于具有新[!DNL Quality Patches Tool]发行版本的其他版本。 要检查修补程序是否与您的Adobe Commerce版本兼容，请将`magento/quality-patches`包更新到最新版本，并在[[!DNL Quality Patches Tool]：搜索修补程序页面](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html?lang=zh-Hans)上检查兼容性。 使用修补程序ID作为搜索关键字来查找修补程序。

## 问题

Fotorama库无法正确加载，从而导致仅显示第一个附加的图像。

<u>重现步骤</u>：

1. 创建一个简单的产品并将多个图像附加到该产品。
1. 打开店面上的产品。

<u>预期的结果</u>：

图像库显示所有附加的图像。

<u>实际结果</u>：

1. 只有第一个图像可见。
1. 浏览器控制台显示以下错误：

```
Javascript error(severe) - http://magento.com/static/version1717863530/frontend/magento/luma/en_us/mage/gallery/gallery.js 309:29 uncaught type error: $fotorama element.fotorama is not a function
```
