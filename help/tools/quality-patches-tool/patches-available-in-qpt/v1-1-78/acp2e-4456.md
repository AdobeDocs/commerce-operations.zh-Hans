# ACP2E-4456：取消带有GraphQL突变的订单不会将完全使用礼品卡支付的订单移至“已关闭”状态

ACP2E-4456修补程序修复了GraphQL订单取消功能无法将礼品卡支付订单过渡到[!UICONTROL Closed]状态的问题。 安装[[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.78时，此修补程序可用。 修补程序ID为ACP2E-4456。 请注意，此问题计划在Adobe Commerce 2.4.9中修复。

## 受影响的产品和版本

**为Adobe Commerce版本创建了修补程序：**

* Adobe Commerce（所有部署方法） 2.4.8-p3

**与Adobe Commerce版本兼容：**

* Adobe Commerce（所有部署方法） 2.4.8 - 2.4.8-p4

>[!NOTE]
>
>该修补程序可能适用于具有新[!DNL Quality Patches Tool]发行版本的其他版本。 要检查修补程序是否与您的Adobe Commerce版本兼容，请将`magento/quality-patches`包更新到最新版本，并在[[!DNL Quality Patches Tool]：搜索修补程序页面](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html?lang=zh-Hans)上检查兼容性。 使用修补程序ID作为搜索关键字来查找修补程序。

## 问题

通过GraphQL突变取消订单不会将礼品卡支付订单过渡到[!UICONTROL Closed]状态的问题。

<u>重现步骤</u>：

1. 创建任何类型的礼品卡。
1. 创建一个简单的产品。
1. 启用&#x200B;**[!UICONTROL Free Shipping]**。
1. 导航到&#x200B;**[!UICONTROL Stores]** > **[!UICONTROL Configuration]** > **[!UICONTROL Sales]** > **[!UICONTROL Order cancellation]**，并将&#x200B;**[!UICONTROL Order cancellation through GraphQL]**&#x200B;设置为&#x200B;*是*，以启用GraphQL订单取消功能。
1. 登录到店面帐户。
1. 购买礼品卡，生成订单发票，并记下礼品卡编号。
1. 将简单产品添加到购物车。
1. 在结账期间选择&#x200B;**[!UICONTROL Free Shipping]**。
1. 使用礼品卡完成付款。
1. 为订单开票。
1. 使用以下GraphQL突变取消订单：

   ```
   mutation {
   	cancelOrder(input: { order_id: "Nw==", reason: "Other" }) {
   		error
   		order {
   			carrier
   			created_at
   			email
   			gift_receipt_included
   			grand_total
   			id
   			increment_id
   			number
   			order_date
   			shipping_method
   			status
   		}
   	}
   }
   ```

<u>预期的结果</u>：

订单状态更改为“已取消”。

<u>实际结果</u>：

未取消订单。
