# ACSD-67908：在多存储设置中，JavaScript文件无法正确合并

ACSD-67908修补程序修复了JavaScript文件在多存储设置中无法正确合并的问题。 安装[[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.73时，此修补程序可用。 修补程序ID为ACSD-67908。 请注意，此问题计划在Adobe Commerce 2.4.9中修复。

>[!NOTE]
>
>Adobe不建议使用&#x200B;**[!UICONTROL Merge JavaScript Files]**。 在应用此修补程序之前，请尝试禁用JavaScript合并并重新部署静态内容以确认问题是否已解决。 有关详细信息，请参阅[优化资源文件](/help/implementation-playbook/best-practices/development/optimize-css-js-files.md#merge-js-files)。


## 受影响的产品和版本

**为Adobe Commerce版本创建了修补程序：**

* Adobe Commerce（所有部署方法） 2.4.7-p5

**与Adobe Commerce版本兼容：**

* Adobe Commerce（所有部署方法） 2.4.4 - 2.4.8-p3

## 问题

在多存储设置中，合并的JavaScript文件无法正确生成，从而导致文件损坏和店面错误。

<u>重现步骤</u>：

1. 设置多个网站、商店和商店视图。
1. 设置生产模式并跳过编译：

   ```shell
   bin/magento deploy:mode:set production --skip-compilation
   ```

1. 部署生成的代码：

   ```shell
   bin/magento setup:di:compile
   ```

1. 使用紧凑策略部署静态内容：

   ```shell
   bin/magento setup:static-content:deploy --strategy compact --jobs 10
   ```

1. 删除合并缓存文件夹的内容：

   ```text
   rm -rf pub/static/_cache/merged/*
   ```

1. 清除缓存：

   ```shell
   bin/magento cache:clean
   ```

1. 在单独的浏览器选项卡或窗口中同时打开所有商店视图。


<u>预期的结果</u>：

所有JavaScript文件合并成功，店面运行正常，没有错误。

<u>实际结果</u>：

某些合并的JavaScript文件已损坏，且开头不是正确的[!DNL RequireJS]模块声明。
