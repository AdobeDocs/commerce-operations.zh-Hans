# ACSD-68206： [!DNL GraphQL]服务器因速率限制和PHP Redis扩展而失败

ACSD-68206修补程序修复了在使用速率限制功能和PHP Redis扩展时GraphQL应用程序服务器引发错误，进而中断速率限制功能的问题。 安装[[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.73时，此修补程序可用。 修补程序ID为ACSD-68206。 请注意，此问题计划在Adobe Commerce 2.4.9中修复。

## 受影响的产品和版本

**为Adobe Commerce版本创建了修补程序：**

* Adobe Commerce（所有部署方法） 2.4.8-p2

**与Adobe Commerce版本兼容：**

* Adobe Commerce（所有部署方法） 2.4.7 - 2.4.8-p3

>[!NOTE]
>
>该修补程序可能适用于具有新[!DNL Quality Patches Tool]发行版本的其他版本。 要检查修补程序是否与您的Adobe Commerce版本兼容，请将`magento/quality-patches`包更新到最新版本，并在[[!DNL Quality Patches Tool]：搜索修补程序页面](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html)上检查兼容性。 使用修补程序ID作为搜索关键字来查找修补程序。

## 问题

当GraphQL应用程序服务器使用速率限制功能并安装PHP Redis扩展时，会出现此问题，从而导致速率限制功能受损。

<u>重现步骤</u>：

1. 安装PHP Redis扩展。
1. 为应用程序配置Redis。
1. 设置费率限制器，以允许每分钟为每位客户订购一单。
1. 启动[!DNL GraphQL]应用程序服务器。
1. 作为客户，使用`PlaceOrder` [!DNL GraphQL]突变提交订单。
1. 在一分钟内，为同一客户使用`PlaceOrder` [!DNL GraphQL]突变提交第二份订单。

<u>预期的结果</u>：

`PlaceOrder` [!DNL GraphQL]请求返回消息限制订单尝试。

<u>实际结果</u>：

对于第二个PlaceOrder GraphQL请求，服务器返回500错误。 异常日志记录：

```
report.CRITICAL: Error: Call to a member function exec() on null in vendor/colinmollenhour/credis/Client.php:1336
```
