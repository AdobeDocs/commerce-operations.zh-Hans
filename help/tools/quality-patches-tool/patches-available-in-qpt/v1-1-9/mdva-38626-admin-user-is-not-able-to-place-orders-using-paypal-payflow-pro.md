# MDVA-38626：管理员用户无法使用PayPal Payflow Pro下订单

MDVA-38626修补程序解决了管理员用户无法使用PayPal Payflow Pro支付方式在后端下订单的问题。 安装[Quality Patches Tool (QPT)](https://experienceleague.adobe.com/zh-hans/docs/commerce-operations/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches) 1.1.9时，此修补程序可用。 修补程序ID为MDVA-38626。 请注意，该问题计划在Adobe Commerce 2.4.5中修复。

## 受影响的产品和版本

**为Adobe Commerce版本创建了修补程序：**

* Adobe Commerce（所有部署方法） 2.4.2

**与Adobe Commerce版本兼容：**

* Adobe Commerce（所有部署方法） 2.3.3 - 2.4.3-p1

>[!NOTE]
>
>该修补程序可能适用于具有新的Quality Patches Tool版本的其他版本。 要检查修补程序是否与您的Adobe Commerce版本兼容，请将`magento/quality-patches`包更新到最新版本，并在[[!DNL Quality Patches Tool]：搜索修补程序页面](https://experienceleague.adobe.com/zh-hans/docs/commerce-operations/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches)上检查兼容性。 使用修补程序ID作为搜索关键字来查找修补程序。

## 问题

管理员用户无法使用PayPal Payflow Pro支付方式在后端下单。

<u>重现步骤</u>：

1. 配置PayPal Payflow Pro支付方式。
1. 转到&#x200B;**客户** > **所有客户**&#x200B;并创建客户帐户。
1. 单击&#x200B;**创建订单**。
1. 将任意项目添加到购物车。
1. 尝试使用Payflow Pro下订单。

<u>预期的结果</u>：

已下订单。

<u>实际结果</u>：

用户收到500错误。 报表日志包含：

```
{"0":"No such entity with cartId = 0","1":"#1 Magento\\Quote\\Model\\QuoteRepository->loadQuote() called at [app\/code\/Magento\/Quote\/Model\/QuoteRepository.php:136]\n#2 Magento\\Quote\\Model\\QuoteRepository->get() called at [lib\/internal\/Magento\/Framework\/Interception\/Interceptor.php:58]\n#3 Mag
```
