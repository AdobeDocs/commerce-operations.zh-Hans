# ACP2E-4552： GraphQL响应不返回公司状态

ACP2E-4552修补程序修复了GraphQL响应中未返回公司状态的问题。 安装[[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.80时，此修补程序可用。 修补程序ID为ACP2E-4552。 请注意，此问题计划在Adobe Commerce B2B 1.5.2-p3中修复。

## 受影响的产品和版本

**为Adobe Commerce版本创建了修补程序：**

* Adobe Commerce（所有部署方法）B2B 1.5.2-p3

**与Adobe Commerce版本兼容：**

* Adobe Commerce（所有部署方法）B2B >=1.5.0 &lt;1.5.3

>[!NOTE]
>
>该修补程序可能适用于具有新[!DNL Quality Patches Tool]发行版本的其他版本。 要检查修补程序是否与您的Adobe Commerce版本兼容，请将`magento/quality-patches`包更新到最新版本，并在[[!DNL Quality Patches Tool]：搜索修补程序页面](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html?lang=zh-Hans)上检查兼容性。 使用修补程序ID作为搜索关键字来查找修补程序。

## 问题

GraphQL响应不会返回公司状态。

<u>先决条件</u>：

[!DNL B2B]已安装。

<u>重现步骤</u>：

1. 在管理员中，转到&#x200B;**[!UICONTROL Stores]** > *[!UICONTROL Settings]* > **[!UICONTROL Configuration]** > **[!UICONTROL General]** > **[!UICONTROL B2B Features]**，并将&#x200B;**[!UICONTROL Enable Company]**&#x200B;设置为`Yes`。
1. 以管理员身份登录Commerce管理员。
1. 创建三个具有以下状态的公司：“有效”、“已拒绝”和“待审批”。
1. 在“管理员”中，转到&#x200B;**[!UICONTROL Customers]** > **[!UICONTROL All Customers]**。 打开客户帐户，并从&#x200B;**[!UICONTROL Associate to Company]**&#x200B;列表中，将客户分配给每个创建的公司。
1. 发送以下GraphQL请求：

   ```
   {
     customer {
       firstname
       lastname
       email
       status
       companies {
        items {
           id
           legal_name
           name
           status
        }
       }
     }
   }
   ```

<u>预期的结果</u>：

GraphQL响应会为客户关联的所有公司返回正确的状态值（例如，“活动”、“未决批准”、“已拒绝”和“已阻止”）。 也可以按公司状态对响应进行筛选或排序。

<u>实际结果</u>：

GraphQL响应不包括与客户关联的公司的公司状态值。
