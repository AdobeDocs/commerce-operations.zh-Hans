# ACP2E-4626：Storefront JavaScript文件加载两次，导致重复加载和不稳定行为

ACP2E-4626修补程序修复了两次请求和执行店面JavaScript文件的问题，该问题会导致间歇性的重复加载和不稳定的行为。修复后，JavaScript依赖项仅加载一次，可在保留现有mixin功能的同时消除重复请求。 安装[[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.80时，此修补程序可用。 修补程序ID为ACP2E-4626。 请注意，此问题计划在Adobe Commerce 2.4.9中修复。

## 受影响的产品和版本

**为Adobe Commerce版本创建了修补程序：**

* Adobe Commerce（所有部署方法） 2.4.8-p3

**与Adobe Commerce版本兼容：**

* Adobe Commerce（所有部署方法） 2.4.7 - 2.4.8-p5

>[!NOTE]
>
>该修补程序可能适用于具有新[!DNL Quality Patches Tool]发行版本的其他版本。 要检查修补程序是否与您的Adobe Commerce版本兼容，请将`magento/quality-patches`包更新到最新版本，并在[[!DNL Quality Patches Tool]：搜索修补程序页面](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html)上检查兼容性。 使用修补程序ID作为搜索关键字来查找修补程序。

## 问题

某些Storefront JavaScript文件被请求并执行两次的问题，会导致间歇性的重复加载和不稳定的行为。 修复后，依赖项仅加载一次，可在保留现有mixin功能的同时消除重复请求。”

<u>先决条件</u>：

1. 已安装MSI。

<u>重现步骤</u>：

1. 在浏览器中打开任何店面页面。
1. 打开浏览器&#x200B;**[!UICONTROL DevTools]**&#x200B;并选择&#x200B;**[!UICONTROL Console]**&#x200B;选项卡。
1. 运行以下命令。

   ```
   require.config({ deps: ['mage/common'] });
   ```

<u>预期的结果</u>：

不发生重复加载。 调用将返回所需的函数，并且不会生成任何新的脚本请求。

<u>实际结果</u>：

新请求显示在&#x200B;**[!UICONTROL Network]**&#x200B;选项卡中（例如，mage/common.js、requirejs/domReady.js、jquery.js）。
