# ACP2E-4653：[!DNL REST] API中缺少购物车价格规则类别范围条件

ACP2E-4653修补程序修复了在通过[!DNL REST] API检索或更新规则时，未显示&#x200B;**[!UICONTROL Category (Parent Only)]**&#x200B;和&#x200B;**[!UICONTROL Category (Children Only)]**&#x200B;的购物车价格规则条件属性范围的问题。 安装[[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.80时，此修补程序可用。 修补程序ID为ACP2E-4653。 请注意，此问题计划在Adobe Commerce 2.4.9中修复。

## 受影响的产品和版本

**为Adobe Commerce版本创建了修补程序：**

* Adobe Commerce（所有部署方法） 2.4.8-p2

**与Adobe Commerce版本兼容：**

* Adobe Commerce（所有部署方法） 2.4.8 - 2.4.8-p5

>[!NOTE]
>
>该修补程序可能适用于具有新[!DNL Quality Patches Tool]发行版本的其他版本。 要检查修补程序是否与您的Adobe Commerce版本兼容，请将`magento/quality-patches`包更新到最新版本，并在[[!DNL Quality Patches Tool]：搜索修补程序页面](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html)上检查兼容性。 使用修补程序ID作为搜索关键字来查找修补程序。

## 问题

当通过[!DNL REST] API检索或更新购物车价格规则时，**[!UICONTROL Category (Parent Only)]**&#x200B;和&#x200B;**[!UICONTROL Category (Children Only)]**&#x200B;的属性范围未公开。

<u>重现步骤</u>：

1. 使用管理员凭据向`rest/V1/integration/admin/token`发送`POST`请求以生成管理员令牌。
1. 通过使用[!DNL REST]终结点`rest/V1/salesRules`和使用`category_ids`属性的操作条件来创建购物车价格规则。
1. 在“管理”面板中，打开购物车价格规则。 在&#x200B;**[!UICONTROL Actions]**&#x200B;部分的&#x200B;**[!UICONTROL Apply the rule only to cart items matching the following conditions (leave blank for all items]**&#x200B;下，手动添加&#x200B;**[!UICONTROL Category (Children Only)]**&#x200B;条件。
1. 设置条件，使&#x200B;**[!UICONTROL Category (Children Only)]**&#x200B;不是`<category_id>`。
1. 通过向`rest/V1/salesRules/<rule_id>`发送`GET`请求，通过API检索规则。
1. 观察响应中返回的`action_condition`值。

<u>预期的结果</u>：

1. 应该可以通过[!DNL REST] API创建、检索和更新具有&#x200B;**[!UICONTROL Category (Children Only)]**&#x200B;条件的购物车价格规则。
1. 应显式公开属性作用域，以便正确区分类别、**[!UICONTROL Category (Parent Only)]**&#x200B;和&#x200B;**[!UICONTROL Category (Children Only)]**。 例如，`action_condition`应按如下方式返回：

   ```
           "action_condition": {
               "condition_type": "Magento\\SalesRule\\Model\\Rule\\Condition\\Product\\Combine",
               "conditions": [
                   {
                       "condition_type": "Magento\\SalesRule\\Model\\Rule\\Condition\\Product",
                       "operator": "!=",
                       "attribute_name": "category_ids",
                       "value": "4",
                       "extension_attributes": {
                           "attribute_scope": ""
                       }
                   },
                   {
                       "condition_type": "Magento\\SalesRule\\Model\\Rule\\Condition\\Product",
                       "operator": "==",
                       "attribute_name": "category_ids",
                       "value": "5",
                       "extension_attributes": {
                           "attribute_scope": "children"
                       }
                   }
               ],
               "aggregator_type": "all",
               "operator": null,
               "value": "1"
           },
   ```

<u>实际结果</u>：

1. 两个条件属性名称都作为`category_ids`返回。
1. 不清楚如何通过[!DNL REST] API创建或更新具有&#x200B;**[!UICONTROL Category (Children Only)]**&#x200B;条件的规则。
