# ACSD-68499：对于超出可用库存的数量，GraphQL updateCartItems突变返回&#x200B;*success*

ACSD-68499修补程序修复了以下问题：更新数量超过可用库存量时，GraphQL `updateCartItems`突变返回错误的&#x200B;*success*&#x200B;响应，从而导致数量和总数虚增。 安装[[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.74时，此修补程序可用。 修补程序ID为ACSD-68499。 请注意，此问题计划在Adobe Commerce 2.4.9中修复。

## 受影响的产品和版本

**为Adobe Commerce版本创建了修补程序：**

* Adobe Commerce（所有部署方法） 2.4.8-p2

**与Adobe Commerce版本兼容：**

* Adobe Commerce（所有部署方法） 2.4.4 - 2.4.8-p3

>
>该修补程序可能适用于具有新[!DNL Quality Patches Tool]发行版本的其他版本。 要检查修补程序是否与您的Adobe Commerce版本兼容，请将`magento/quality-patches`包更新到最新版本，并在[[!DNL Quality Patches Tool]：搜索修补程序页面](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html?lang=zh-Hans)上检查兼容性。 使用修补程序ID作为搜索关键字来查找修补程序。

## 问题

更新超出可用库存的数量时，Adobe Commerce中的GraphQL `updateCartItems`突变错误地返回&#x200B;*success*&#x200B;响应，导致购物车总数和数量虚增。

<u>重现步骤</u>：

1. 创建一个空购物车。
1. 将库存有限的产品（例如，10件可用）添加到购物车。
1. 查询购物车并记下`cart_item_id`。
1. 使用以下突变将购物车物料数量更新为100（超出可用库存10）：

   ```
   mutation ($cart: String!, $cartItemId: Int!, $quantity: Float!) {
     updateCartItems(
       input: {
         cart_id: $cart
         cart_items: [{ cart_item_id: $cartItemId, quantity: $quantity }]
       }
     ) {
       cart {
         items {
           product {
             name
           }
           quantity
         }
         prices {
           grand_total {
             value
             currency
           }
         }
       }
     }
   }
   ```

1. 再次查询购物车以验证实际数量。

<u>预期的结果</u>：

库存验证可防止数量更新无效。

<u>实际结果</u>：

`updateCartItems`突变返回&#x200B;*success*&#x200B;响应，其数量为100。
