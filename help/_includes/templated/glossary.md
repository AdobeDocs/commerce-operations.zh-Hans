# Adobe Commerce术语表

## A

### 在折页上方

_形容词_

在浏览器窗口中，加载网页后和用户向下滚动页面之前立即可见的内容。
在设计布局时，请使用灵活的格式来最好地显示此区域中优先级最高的产品、功能、销售、通知、选项等。

使用移动设备和平板电脑时，折页上方的区域会有很大的不同，尤其是在查看时屏幕的大小、尺寸以及方向（纵向与横向）方面。
使用响应式主题和测试可帮助找到正确的内容和布局组合。

_术语属性：_

* _字段：设计_

### 活动分支

_名词_

活动分支或环境是指连接到具有服务访问权限的已部署或正在运行的实例的分支或环境。
停用时，将删除与服务和运行实例的连接，但将保留代码。
它变成了普通的git分支。

_术语属性：_

* _字段：云_

### 适配器

_名词_

一个类，它允许两个其他不兼容的系统协同工作，而无需修改系统的源代码。
示例包括数据库适配器、缓存适配器、文件系统适配器、后处理器库适配器和其他类型的计算适配器。

_术语属性：_

* _字段：编程_

### 管理员

_名词_

在软件中，具有完全管理员权限的用户角色来管理所有功能。
在Adobe Commerce中，管理员用户拥有管理员中所有特性、选项和功能的完全权限和访问权限。
用户还可以创建用户和角色。

了解更多： [添加用户](https://experienceleague.adobe.com/docs/commerce-admin/systems/user-accounts/permissions-users-all.html)

_术语属性：_

* _字段：商务软件_
* _同义词：管理员，超级用户_
* _相关术语：商务管理员_

### 管理区域

_名词_

您商店的受密码保护的后台办公室，管理订单、目录、内容和配置。
用户可访问“管理员”区域来管理商店，包括产品、订单、发货、CMS内容、店面设计、客户信息等。
管理员用户具有与权限关联的角色，这些权限控制对特性、选项和功能的访问。

了解更多： [Adobe Commerce用户指南](https://experienceleague.adobe.com/docs/commerce-admin/user-guides/home.html)

_术语属性：_

* _字段：商务软件_
* _同义词：管理员，管理面板，后端，管理界面，管理员UI_
* _相关术语：管理员_

### 管理员变量

_名词_

### 面积

_名词_

区域是Magento应用范围的抽象术语。
区域是组织代码以优化请求处理的逻辑组件。
区域减少了从店面访问配置对象的内存需求，并通过仅加载所需的相关代码来简化Web服务调用。
每个区域都可以包含完全不同的代码来处理URL和请求。

Adobe Commerce地区包括：

* 管理员(adminhtml)
* 店面
* Web API REST(webapi_rest)

_术语属性：_

* _字段：商务软件_
* _相关术语：商务组件，storefront_

### 属性

_名词_

描述产品某些方面的产品的特征或属性。
Adobe Commerce或Magento Open Source用户可以创建自定义属性以添加到默认属性集或自定义属性集。
通过管理员或编程方式创建这些属性。
示例：颜色、大小、重量、价格、年龄、性别等。

自定义属性是一种实体属性值(EAV)属性类型。

对于Google购物广告渠道和AmazonSales Channel等集成，您可以将商务属性映射到第三方中的属性，以便正确显示和销售产品、展示广告。

了解更多： [EAV和扩展extension_attributes](https://developer.adobe.com/commerce/php/development/components/attributes/)

_术语属性：_

* _字段：商务软件，产品_
* _同义词：产品属性，自定义属性_
* _相关术语：扩展属性_

### 目录

_名词_

对于商户，目录表示其产品库存。
在Adobe Commerce架构中，目录由类别、产品和产品属性组成。

每个商务商店只有一个产品目录，所有商店视图都共享该目录。

### 可配置产品

_名词_

可配置产品看起来像单个产品，其中包含每个变体的选项下拉列表。
实际上，每个选项都是一个具有唯一SKU的单独简单产品，这样就可以跟踪每个产品变体的库存。
为了达到类似的效果，可以将简单的产品与自定义选项一起使用，但无法跟踪每个变体的库存。
具有多个选项的产品有时称为复合产品。

尽管可配置产品使用的SKU较多，且最初设置所花费的时间可能较长，但最终却可以节省您的时间。
如果您计划扩展业务，那么对于具有多个选项的产品，可配置的产品类型可能是更好的选择。

示例：有两种颜色和三种尺寸的T恤。
必须将六个变体创建为单个产品（每个产品都有其自己的SKU）。
然后，所有变体都会添加到可配置产品中，供客户选择大小和颜色，然后将其添加到购物车。

_术语属性：_

* _字段：商务软件，产品_
* _相关术语：产品类型_

## U

### UI组件

_名词_

专为Adobe Commerce软件设计的标记，用于实现更简单、更灵活的用户界面(UI)渲染。
UI组件系统的目标包括：

* 简化布局处理XML文件

* 允许从较小的组件中创建更复杂的UI组件
* 将UI组件的数据预渲染为JSON，并与后端数据对象紧密绑定
* 使用AJAX更新组件数据
* 引入新DSL以创建上述项目

了解更多： [UI组件指南](https://developer.adobe.com/commerce/frontend-core/ui-components/), [页面生成器](https://experienceleague.adobe.com/docs/commerce-admin/page-builder/introduction.html)

### 向上

_名词_

[PWA Studio](https://github.com/magento/pwa-studio) 使用 [向上](https://developer.adobe.com/commerce/pwa-studio/guides/packages/upward/) 开发。
UPPRAD是统一渐进式Web应用程序响应定义的首字母缩略词。
向上定义文件描述Web服务器如何传送和支持Progressive Web Application。

向上定义文件使用与平台无关的声明性语言提供有关服务器行为的详细信息。
这允许Progressive Web Application在任何技术堆栈上以任何语言在符合UPPRAD的服务器上运行，因为应用程序只关注来自UPPRAD服务器的HTTP端点行为。

向上服务器使用定义文件来确定从应用程序外壳发出请求的适当过程或服务。
它描述了服务器应如何处理请求并为其构建响应。

PWA项目可以包含UPPRAD定义文件以指定其服务依赖项。

_术语属性：_

* _字段：设计，商业软件，编程_
* _同义词：PWA Studio、统一的渐进式Web应用程序响应定义_
* _相关术语：pwa_

### 虚拟产品

_名词_

表示可销售的非实物产品，如会员资格、服务、保修或订购。
虚拟产品可以单独销售，也可以作为以下产品类型的一部分进行销售：分组产品和捆绑产品。
不需要装运或库存。

创建虚拟产品和简单产品的过程几乎是一样的。

### 虚拟类型

_名词_

虚拟类型是一种在不影响其他类、不必创建类文件的情况下，将不同的依赖项注入到现有PHP类中的方法。
虚拟类型只能由 `<type>` i.xml文件中的元素，源代码中的元素则从不。
它们不能扩展，也不能作为类构造函数中的依赖项进行引用。

* _字段：编程_
* _相关术语：php_
